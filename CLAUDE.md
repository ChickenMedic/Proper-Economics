# ProperEconomics.com

A free, layman-friendly guide to the history of economic thought: plain-English economist profiles, an interactive timeline, and hands-on simulations. The full product brief lives in `proper-economics-brief_1.md` — read it before making product decisions.

## Stack

- **Next.js (App Router) with full static export** (`output: 'export'` in `next.config.ts`). No SSR, no backend, no API routes. Everything is static HTML + client-side interactivity.
- **TypeScript throughout.** Strict mode.
- **Tailwind CSS v4** (CSS-first config in `src/app/globals.css` via `@theme`). Light and dark mode via `prefers-color-scheme` plus a class toggle.
- **Content is MDX in-repo** under `/content` — Git is the CMS. No CMS, no database.
- **Interactives are bespoke React + SVG** (d3-scale for scales only; React owns the DOM). No charting libraries.
- **Search** is client-side over a build-time JSON index (`public/search-index.json`, generated by `scripts/build-indexes.mjs`).

## Commands

- `npm run dev` — dev server (runs the index build first)
- `npm run build` — generates indexes, then static export to `out/`
- `npm run lint` — ESLint
- `npm run typecheck` — `tsc --noEmit`
- `node scripts/build-indexes.mjs` — regenerates `public/search-index.json` and `src/data/timeline.json` from MDX frontmatter. **Run this after adding/editing content files.**

## Layout

```
content/
  economists/*.mdx    Profile pages — frontmatter schema in src/lib/content.ts
  schools/*.mdx       School-of-thought explainers
  glossary/*.mdx      One term per file; powers site-wide tooltips
src/
  app/                Routes (see brief §4 for the full map)
  components/         Shared UI; interactives live in components/interactives/
  lib/                Content loading (content.ts), types, utilities
  data/               Build-generated JSON (timeline.json) + hand-maintained data (events, school bands)
scripts/build-indexes.mjs   Build-time index generation
amplify.yml           AWS Amplify build spec (install → lint → typecheck → build)
```

## Content rules (non-negotiable, from the brief)

1. **Plain English first** — explainable to a smart 15-year-old; jargon gets a glossary tooltip (`<G term="...">` in MDX) on first use. Flesch-Kincaid grade 8–10 for summaries.
2. **Steelman everyone.** Exposition before critique, always. Profile sections in fixed order: The idea in one paragraph → The world they lived in → What they argued → Where it breaks down → Lasting influence → The conversation → Try it → Further reading.
3. **Critiques cite who makes them; contested claims are labeled contested.**
4. **Verify facts, don't recall them** — biographical dates, Nobel years, and especially tax data must be checked against sources (Britannica, SEP, OECD, Tax Foundation), never written from model memory alone. Tax figures carry a "data as of" year and per-country source links.
5. **Portraits are public-domain (Wikimedia Commons)** with source/license recorded in frontmatter.
6. **Voice:** warm, direct, a little wry — a favorite teacher, not a textbook. Concrete everyday examples before abstractions. Rounded, contextualized numbers.

## Frontmatter schema (economists)

See `src/lib/content.ts` for the typed source of truth. Key fields: `name`, `slug`, `born`, `died`, `nationality`, `school` (slug), `era` (one of the eight era ids in `src/data/eras.ts`), `knownFor` (one line), `portrait` {src, source, license}, `influencedBy`/`influenced`/`arguedAgainst` (arrays of economist slugs), `works` [{title, year, url, free}], `tryIt` (module slug), `flagship` (bool), `tags`.

## Engineering rules

- **Accessibility is a launch requirement**, not polish: every interactive needs keyboard control (real `<input type="range">` sliders), a text alternative describing what it demonstrates, and an explainer panel that narrates state changes.
- Interactives lazy-load below the fold (`next/dynamic`). Keep profile-page JS < 200 KB gzipped; Lighthouse ≥ 90 target on all categories.
- Mobile-first: interactives work with touch; the timeline degrades to a vertical era list on small screens.
- No user accounts, no cookies needing consent banners, no localStorage dependencies, no third-party embeds or analytics without asking the owner (Sam Dawson) first.
- Cross-links between profiles use economist slugs; the build-index script fails the build on dangling references — keep it that way.

## Deployment

GitHub → AWS Amplify Hosting. `main` → production (propereconomics.com), PR previews for branches. `amplify.yml` runs install, lint, typecheck, build; a failing check blocks deploy. The static export lands in `out/`.
